Before Contacting Support

Please verify the following first:

  • Aruba IAP cluster is running AOS 8 (versions 8.10-8.13) — check with show version
  • Your AP model is AP-3xx, AP-5xx, or AP-6xx series
  • REST API is enabled on your Virtual Controller (allow-rest-api + commit apply)
  • You can reach the VC IP from your iOS/macOS device on port 4343 (HTTPS)
  • Your credentials work via the VC web interface first

Frequently Asked Questions

What devices are supported?

iOS 16+ and macOS Tahoe 26.3.1 or later.

Does this work with HPE InstantOn?

No. NoCentral only works with Aruba IAP clusters running AOS 8 (versions 8.10-8.13). HPE InstantOn uses a completely different API and is not supported.

Does this work with Campus Controllers or Mobility Conductors?

Not yet. Current version supports Aruba Instant APs (IAP) only. Campus Controller support is planned for a future release with no ETA.

Do I need an Aruba Central subscription?

No. NoCentral works directly with your Virtual Controller over your local network without any cloud dependencies.

How do I connect to my Virtual Controller?

Enter your VC's IP address, username, and password. NoCentral will securely store credentials in your system Keychain. Default credentials are typically admin / admin unless you've changed them.

Common Issues

Can't connect to Virtual Controller

  • Verify REST API is enabled: allow-rest-api then commit apply
  • Check that VC IP is reachable on port 4343 (HTTPS)
  • Self-signed certificates are handled automatically — no manual trust needed
  • Ensure you're connecting to the master IAP (look for * next to IP in show aps)

Login fails with correct credentials

  • Test credentials via VC web UI first (https://VC-IP:4343)
  • Sessions expire after 15 minutes of inactivity
  • If you changed the password recently, delete and re-add the site in NoCentral

Missing APs or incomplete data

  • Connect to the VC IAP, not a member AP
  • During cluster elections, some data may be temporarily unavailable
  • Offline APs won't show real-time stats

App crashes or freezes

  • Force quit and restart the app
  • Verify your iOS/macOS version meets the minimum requirements
  • Very large clusters (100+ APs) may experience slower load times

Known Limitations

Triband APs: AP-6xx and AP-7xx series with three radios currently only show radio0 and radio1 statistics. Radio2 support is not yet implemented.
  • Large clusters (100+ APs): Some views may load slowly
  • Campus Controllers: Not supported (IAP only)
  • Mobility Conductors: Not supported (IAP only)
  • AOS 8.13 is the final version of AOS 8 — future compatibility depends on whether HPE releases AOS 9 or EOLs the platform
